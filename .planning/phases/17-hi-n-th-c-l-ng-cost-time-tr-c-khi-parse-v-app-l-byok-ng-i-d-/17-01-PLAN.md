## Task 2 — `estimateParseRun.ts`

<action>
Add `src/lib/ai/estimateParseRun.ts` exporting:

**Types**

- `ParseRunEstimateInput` — fields: `pageCount: number | null`, `extractedTextCharCount: number`, `parseStrategy: ParseStrategy` (import type from `@/lib/ai/parseLocalStorage`), `enableOcr: boolean`, `attachPageImage: boolean`.

- `ParseRunEstimate` — fields at minimum:
  - `cappedPages: number` — `min(max(pageCount,1), VISION_MAX_PAGES_DEFAULT)` when `pageCount` known and &gt; 0; when `pageCount` null or ≤0 use `1` as pessimistic placeholder for calls that need a numeric base (document in JSDoc).
  - `visionApiCalls: number`
  - `chunkParseApiCallsUpperBound: number` — 0 when policy resolves to vision-only path for whole run (Accurate, or Fast/Hybrid with OCR off); else `min(cappedPages, 60)`.
  - **Token upper bounds (heuristic, no API):** export named constants `HEURISTIC_VISION_INPUT_TOKENS_PER_STEP`, `HEURISTIC_VISION_OUTPUT_TOKENS_PER_STEP`, `HEURISTIC_CHUNK_INPUT_TOKENS_UPPER`, `HEURISTIC_CHUNK_OUTPUT_TOKENS_UPPER` (integers, documented as rough). Fields: `estimatedVisionInputTokensUpper`, `estimatedVisionOutputTokensUpper` (= `visionApiCalls ×` respective constants, include **fallback** vision steps in same totals when `visionFallbackApiCallsUpperBound` &gt; 0 — add both to vision totals); `estimatedChunkInputTokensUpper`, `estimatedChunkOutputTokensUpper` (= `chunkParseApiCallsUpperBound ×` chunk constants). If chunk calls 0, token fields for chunk are `0`.
  - `visionFallbackApiCallsUpperBound: number` — same formula as primary vision when chunk path may fall back (see strategy rules below); `0` when no fallback path.
  - `estimatedDurationSecondsMin` and `estimatedDurationSecondsMax` (integers) from sums of heuristics (e.g. `8000` ms per vision step, `4000` ms per chunk upper bound — use **named exported constants** `HEURISTIC_MS_PER_VISION_STEP` and `HEURISTIC_MS_PER_CHUNK_UPPER` at top of file).
  - `disclaimer: string` — must equal exactly: `Estimates are not billing guarantees; actual tokens depend on your model and PDF content.`

**Logic**

1. Import `decideParseRoute` and call with `{ pageCount, extractedTextCharCount, parseStrategy, enableOcr }`.
2. Compute `cappedPages` from `pageCount` and `VISION_MAX_PAGES_DEFAULT`.
3. **Vision calls** for a full vision-only parse of capped pages:
   - If `attachPageImage`: `visionApiCalls = cappedPages`
   - Else if `cappedPages === 1`: `visionApiCalls = 1`
   - Else: `visionApiCalls = cappedPages - 1`
4. **Strategy-specific** (align with `AiParseSection` routing):
   - **`accurate`:** use full vision formula above for `visionApiCalls`; `chunkParseApiCallsUpperBound = 0`; duration from vision steps only (+ optional small constant for PDF render e.g. `2000` ms once — export `HEURISTIC_MS_RENDER_ONCE`).
   - **`fast` / `hybrid`:** If `decideParseRoute.executionFamily === "layout_chunk"` and `enableOcr`: set `chunkParseApiCallsUpperBound = min(cappedPages, 60)`; primary `visionApiCalls = 0` for the **happy path**; set `visionFallbackApiCallsUpperBound` to the **full** vision-only step count (same attach/single/pair formula as Accurate on `cappedPages`) so token totals include `visionApiCalls + visionFallbackApiCallsUpperBound` for vision-token upper fields. If OCR off → same as accurate (`chunkParseApiCallsUpperBound = 0`, `visionFallbackApiCallsUpperBound = 0`, `visionApiCalls` from full vision formula).
   - If hybrid path would need OCR gate detail not in pure input, treat same as fast for counts (document limitation in JSDoc).

5. Export `export function estimateParseRun(input: ParseRunEstimateInput): ParseRunEstimate`.

No `console`, no side effects.
</action>
