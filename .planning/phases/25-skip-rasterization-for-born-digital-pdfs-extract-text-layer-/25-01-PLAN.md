<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PDF bytes → pdf.js | Untrusted user-provided document is parsed in the browser runtime. |
| Policy/logging → UI | Reason codes and rationales flow into user-visible overlay/toasts; must not leak sensitive content. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|----------|----------|-----------|-------------|-----------------|
| T-25-01 | D (DoS) | `sampleTextLayerSignal` | mitigate | Hard-cap sampling to first 3–5 pages, respect AbortSignal, never concatenate full text. |
| T-25-02 | I (Info Disclosure) | `pipelineLog` / rationale strings | mitigate | Log only numeric metrics + file summary; never include extracted PDF text or snippets. |
| T-25-03 | T (Tampering) | Text-layer signal inputs | accept | Inputs are derived locally from pdf.js; attacker can only affect their own session’s routing; fallback to vision when uncertain reduces harmful impact. |
</threat_model>
