Create **`src/lib/ai/embeddingIndexJob.ts`** exporting **`runEmbeddingIndexJob`** with signature along the lines of:

```ts
runEmbeddingIndexJob(options: {
  studySetId: string;
  fullText: string;
  apiKey: string;
  forwardBaseUrl: string;
  embeddingModel?: string;
  signal?: AbortSignal;
  concurrency?: number; // default 2
  onProgress?: (p: EmbeddingIndexJobProgress) => void;
}): Promise<EmbeddingIndexJobResult>;
```

**Behavior (INDEX-34-01, 02, 04, 08):**

1. **Trim + chunk** with `chunkText` (same as today). Empty text → return `{ chunksIndexed: 0, error: "..." }` without calling embed.

2. **Invalidation (INDEX-34-08):** Before writing new chunks, load existing rows (or metadata). If **any** stored row for `studySetId` has `embeddingModel` ≠ resolved model **or** `indexVersion` ≠ `EMBEDDING_INDEX_SCHEMA_VERSION`, call `embeddingIndexClearStudySet(studySetId)` and clear build meta. If metadata says current fingerprint + model + version already match completed build with same `chunkCount` as `chunks.length`, return early with `chunksIndexed: chunks.length` and **no network** (Wave 2 will rely on this — stub ok if meta not yet written in this task, but **hook points** must exist).

3. **Clear + rebuild:** On a real rebuild, `embeddingIndexClearStudySet` then iterate chunks. **Partial index:** write each chunk with `embeddingIndexPut` as today; on failure after retries, return `chunksIndexed` so far + `error` message (do **not** delete successful rows).

4. **Concurrency:** Use a **pool of size `concurrency` (default 2)** — async tasks pulling chunk indices; ensure **deterministic `sourceLabel`** `chunk ${i+1}/${n}``.

5. **Retries:** Wrap per-chunk `embedText` with bounded retries (e.g. max 3) for **429**, **502**, and **TypeError/fetch** failures with exponential backoff (e.g. 300ms × 2^n). Do **not** retry on **401** or obvious schema errors.

6. **Abort:** Respect `signal`; stop scheduling new work; return cancelled result when aborted.

7. **`pipelineLog` (INDEX-34-04):** log `index_job_start`, `index_job_chunk` (optional throttle — at least start/done/cancel/fail), `index_job_done`, `index_job_aborted` with `studySetId` and counts.

8. **Main-thread comment (INDEX-34-03):** File header in `embeddingIndexJob.ts` states **why** work stays on the main thread (same-origin embed route + cookies / simplicity); Workers deferred unless tested.

Refactor **`buildEmbeddingIndexFromPlainText`** to **delegate** to `runEmbeddingIndexJob` (sequential progress compatible) so `RagChunkSearchPanel` and tests keep one code path.

Update **`embeddingIndexPutBuildMeta`** (or equivalent) at successful completion with fingerprint = `sha256Utf8Hex` of **trimmed** full text (reuse `sha256Utf8Hex` from `@/lib/db/parseCacheDb` as in `buildEmbeddingIndex.ts`).
