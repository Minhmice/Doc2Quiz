    Create `src/lib/pdf/pageRoutePlan.ts` exporting:
    - `PageKind = "text" | "bitmap"` (do NOT add a new "rich" detector per D-06; rich layout remains downstream `includePageImage`).
    - `PageTextLayerSignal` for a single page (e.g. `{ pageIndex: number; charCount: number; hasAnyText: boolean }`).
    - `PageRoutePlan` that is serializable and can be passed into `AiParseSection` later. It must contain:
      - `pageCount: number`
      - `pages: Array<{ pageIndex: number; kind: PageKind; reasonCodes: string[] }>` where **every** `pageIndex` in `1..pageCount` appears exactly once (no silent skips).
      - `textPageIndices: number[]` (1-based indices, aligned with pdf.js). This list may be longer than `visionMaxPages` (text lane is not capped by the vision budget).
      - `bitmapPageIndicesAll: number[]` (all bitmap pages, uncapped; 1-based).
      - `bitmapPageIndicesForVision: number[]` (bitmap subset actually eligible for vision; capped to `visionMaxPages`; 1-based).
      - `droppedBitmapPageIndices: number[]` (bitmap pages that are *known* bitmap but excluded from `bitmapPageIndicesForVision` due to `visionMaxPages` cap; deterministic ordering).
      - `droppedBitmapPagesCount: number` (same information as count, for easy logging)
      - `limitsApplied: { previewFirstPageBudget: number; visionMaxPages: number }`
    - A small set of grep-stable reason-code strings similar to `parseRoutePolicy.ts`, such as:
      - `PAGE_TEXT_STRONG`
      - `PAGE_TEXT_WEAK`
      - `PAGE_SIGNAL_UNKNOWN_DEFAULT_BITMAP`
      - `PAGE_SIGNAL_SAMPLED_TEXT_LAYER`
      - `PAGE_DROPPED_VISION_CAP` (page-level reason: bitmap page excluded due to vision cap)

    Create `src/lib/pdf/classifyPdfPages.ts` exporting a single entrypoint:
    - `classifyPdfPages(file, options) -> Promise<PageRoutePlan>`
    - It must:
      - Use pdf.js text layer evidence (getPage(i).getTextContent()) to count chars per page.
      - Never return extracted text strings, only numeric signals.
      - Never throw (return an "all bitmap" conservative plan on failure/abort), mirroring `sampleTextLayerSignal.ts`.
      - Accept caps/budgets as inputs and record them in `limitsApplied`:
        - `previewFirstPageBudget` (3–5, caller-provided)
        - `visionMaxPages` (default from `VISION_MAX_PAGES_DEFAULT`)
      - Apply deterministic thresholds consistent with Phase 25’s doc-level signal (chars/page and non-empty ratio), but at the per-page granularity:
        - A page is "text" if it has >= MIN_CHARS_PER_PAGE_FOR_TEXT_SIGNAL and `hasAnyText=true`.
        - Otherwise "bitmap".
      - **Account for all pages**: always return `pages.length === pageCount`, even if some pages are intentionally not processed downstream due to caps.
      - **Enforce cap only on the bitmap→vision subset**:
        - `bitmapPageIndicesAll` may exceed `visionMaxPages`
        - `bitmapPageIndicesForVision.length` must be `<= visionMaxPages`
        - Any bitmap pages excluded due to cap must be explicitly recorded in `droppedBitmapPageIndices` / `droppedBitmapPagesCount` and tagged with `PAGE_DROPPED_VISION_CAP` (page-level reason codes) so later wiring cannot silently skip them.

    Do NOT import or touch UI modules in this plan (per deliverable: “without changing UI”).
  </action>
