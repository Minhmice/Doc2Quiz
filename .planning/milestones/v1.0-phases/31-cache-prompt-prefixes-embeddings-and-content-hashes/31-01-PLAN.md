  <action>
**`src/lib/ai/parseCacheTypes.ts`**
- Export `ParseCacheLane`: `"vision_batch" | "text_multi_mcq" | "text_single_mcq"`.
- Export `ParseCacheKeyParts`: `{ lane, contentFingerprint, promptIdentity, model, forwardProvider: "openai" | "custom" }` (all strings; `promptIdentity` = hex digest from helper in Task 3).
- Export `ParseCacheRecordMeta`: `{ savedAt: string; lastAccessedAt: string; studySetId?: string }` (optional metadata per D-07 — never part of key).
- Export discriminated union for **stored values** matching lanes: vision → `VisionParseItem[]`; text lanes → `Question[]` (reuse types from `@/types/visionParse`, `@/types/question`).

**`src/lib/db/parseCacheDb.ts`**
- DB name e.g. `doc2quiz-parse-cache`, **DB_VERSION** `1`.
- Two stores: **`vision_batch`** and **`text_chunk`** (multi + single MCQ share this store; **key string must encode lane** so kinds do not collide).
- **Primary key:** `string` = stable concatenation or single SHA-256 over UTF-8 of canonical JSON of `ParseCacheKeyParts` (document canonical field order in comment).
- **Value:** `{ items: ...; meta: ParseCacheRecordMeta }` with JSON serialization; store **`estimatedBytes`** on write for eviction bookkeeping (or compute length of `JSON.stringify`).

**API (async, browser-only):**
- `openParseCacheDb(): Promise<IDBDatabase | null>` — return `null` if `indexedDB` undefined (SSR) or open throws.
- `parseCacheGet(key)`, `parseCacheSet(key, value, meta?)`, `parseCacheTouch` internal on read.
- **`enforceLruAfterSet(storeName)`** — after each set, if over cap, delete oldest by `lastAccessedAt` (index required — add store index `lastAccessedAt` or cursor scan if small cap acceptable for MVP).

**Guards:** Never throw from cache layer into parse pipeline — **log and no-op** on IDB errors so parsing still works without cache (per D-04 baseline).

**CACHE-31-07 evidence:** Repo has no `npm test`. Document **manual** LRU/cap checks in phase **`VALIDATION.md`** (§3). Optional: export a **pure** `canonicalizeParseCacheKeyParts` for future unit tests — not required to ship Phase 31.
