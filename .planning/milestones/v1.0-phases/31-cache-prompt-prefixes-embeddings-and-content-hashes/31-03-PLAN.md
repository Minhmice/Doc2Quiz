**Scope:** Only the **`parseOpenAI`** / OpenAI-compatible forward path used by `parseChunkOnce` and `parseChunkSingleMcqOnce` (not Anthropic-native URLs if still unreachable here — verify `resolveOpenAiCompatEndpointAndModel` usage).

**Key construction (per CACHE-31-02, CACHE-31-03):**
- `lane`: `text_multi_mcq` vs `text_single_mcq`.
- `contentFingerprint`: SHA-256 (or fallback) over **exact `chunkText` / user message bytes** (UTF-8).
- `promptIdentity`: `formatPromptKeyComponent(PROMPTS_BUNDLE_VERSION, await hashPromptIdentity(systemPrompt))` where `systemPrompt` is `MCQ_EXTRACTION_SYSTEM_PROMPT` or `MCQ_SINGLE_CHUNK_SYSTEM_PROMPT`.
- `model`: resolved `modelId`; `forwardProvider`: `"openai" | "custom"` from existing resolution.

**Flow:**
1. Build key; `parseCacheGet` → if hit, return **cloned** `Question[]` / single `Question` (avoid mutation of cached arrays).
2. On miss, call existing forward + parse logic.
3. On success, `parseCacheSet` + LRU.

**Errors:** IDB errors → continue uncached.

**Optional metadata:** Accept `studySetId?: string` on `ParseChunkOnceParams` / `parseChunkSingleMcqOnce` params — pass into `meta` only (D-07); default undefined.

**Do not** cache when `onRawAssistantText` would change behavior? Still cache parsed `Question` results; raw text callback can remain unset for cache hits or re-synthesize minimally — **if callback set, skip cache read/write** for that call to preserve debugging semantics (document in code).
