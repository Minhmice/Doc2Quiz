<task type="auto">
  <name>Task 2: Implement extractText aggregation</name>
  <files>src/lib/pdf/extractText.ts</files>
  <read_first>src/types/pdf.ts, src/lib/pdf/extractText.ts</read_first>
  <action>
Export **`async function extractText(file: File): Promise&lt;ExtractResult&gt;`** that:

1. Calls `pdfjsLib.getDocument({ data: await file.arrayBuffer() }).promise`
2. Iterates `1..numPages`, calls `page.getTextContent()`, concatenates items' `str` with spaces/newlines sensibly
3. Returns `{ text: trimmedString, pageCount: pdf.numPages }`
4. On **pdf.js failure**, **re-throw** or return a result that the UI layer can map to the PDF-04 message (prefer **throwing** a small `Error` with a stable `message` prefix like `PDF_EXTRACT_FAILED` so 01-03 can branch)

For **empty text** after successful parse (`pageCount >= 1` but joined text is empty or whitespace-only): return `{ text: "", pageCount }` so the UI shows the scanned-PDF message per D-06.

Do **not** add OCR. Do **not** call network APIs.
  </action>
  <verify>npm run build</verify>
  <acceptance_criteria>
    - `src/lib/pdf/extractText.ts` contains `export async function extractText`

    - `grep -n "getDocument" src/lib/pdf/extractText.ts` returns at least one match
  </acceptance_criteria>
  <done>extractText compiles and matches ExtractResult</done>
