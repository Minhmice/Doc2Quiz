## Task 1 — Create `parseRoutePolicy.ts`

<action>
Add `src/lib/ai/parseRoutePolicy.ts` exporting:

1. **Types** (exact names flexible but must be exported):
   - Input: `ParseRoutePolicyInput` with fields: `pageCount: number | null`, `extractedTextCharCount: number`, `parseStrategy: "fast" | "accurate" | "hybrid"`, `enableOcr: boolean`.
   - Output: `ParseRouteDecision` with at least: `executionFamily: "vision_pages" | "layout_chunk"`, `reasonCodes: readonly string[]`, `rationale: string` (one or two sentences for UI / logs).

2. **Constants** (at top of file, numeric literals not scattered):
   - `MIN_CHARS_PER_PAGE_FOR_TEXT_SIGNAL = 40` (tunable; document in comment that executor may adjust after UAT).
   - If `pageCount` is null or `<= 0`, treat text signal as **unknown** (do not claim strong native text).

3. **Function** `decideParseRoute(input: ParseRoutePolicyInput): ParseRouteDecision` implementing:
   - **`accurate`:** always `executionFamily: "vision_pages"`, `reasonCodes` includes `"strategy_accurate_vision_first"`.
   - **`fast`:** if `enableOcr` is false → `vision_pages` + code `"ocr_disabled_vision_fallback"`. If `enableOcr` true and strong text signal (`extractedTextCharCount >= pageCount * MIN_CHARS_PER_PAGE_FOR_TEXT_SIGNAL` when `pageCount` known) → `layout_chunk` + `"text_layer_strong"`. Else → `layout_chunk` with `"text_layer_weak_or_unknown"` (still chunk path intent: OCR will run in handler; policy only describes **intent**).
   - **`hybrid`:** mirror `fast` for `executionFamily` selection **or** document in rationale that hybrid uses OCR gating in component — at minimum emit distinct `reasonCodes` from fast (e.g. `"strategy_hybrid"`).

4. **No imports** from `@/components/*`, `react`, or `studySetDb` — only `import type` from `@/lib/ai/parseLocalStorage` for `ParseStrategy` is allowed if you want to alias; otherwise duplicate the union in the input type to avoid cycles.

Do **not** modify `AiParseSection.tsx` in this plan.
</action>
